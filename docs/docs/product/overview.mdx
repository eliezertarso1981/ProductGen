# Produto

ProductGen organiza o trabalho de produto em uma cadeia rastreável:

```text
Estratégia → Discovery → Delivery → Outcomes
```

## Entidades centrais

- `products`: produto em foco dentro do workspace.
- `strategic_pillars`: pilares estratégicos do produto.
- `objectives` e `key_results`: OKRs e métricas.
- `evidences`: sinais brutos vindos de clientes, suporte, vendas, analytics ou pesquisa.
- `pains`: dores agrupadas a partir de evidências.
- `hypotheses`: apostas estruturadas no formato Se/Então/Porque.
- `experiments`: validações das hipóteses.
- `roadmap_items`: iniciativas, épicos e features.
- `outcomes`: medição pós-entrega.

## Princípio de produto

O sistema evita uma fábrica de features: uma entrega só faz sentido quando nasce de uma hipótese rastreável e termina com medição de outcome.
